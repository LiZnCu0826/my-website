# 2026-05-31 工作记录

## 完成的工作：类"疯狂策划"运营模拟游戏完整搭建

### 游戏概念
将经典烹饪后厨玩法置换为游戏策划职场模拟——厨师→策划，食客→玩家群体，做菜→运营动作，满意度→群体好感，店长订单→KPI考核。

### 架构设计
- **Resource 数据层**: PlayerGroupType（4类玩家群体定义）、GameAction（运营动作数据）、KpiDefinition（每日KPI任务）、ActionDatabase/KpiDatabase（数据仓库）
- **Autoload 系统层**: EventBus（全局信号总线）、KpiManager（KPI评估和惩罚管理）、GameManager（核心游戏循环）
- **UI 组件层**: SatisfactionBar（满意度条）、KpiBar（KPI进度条）、ActionCard（可点击运营动作卡牌）、DayStartPanel/DayEndPanel（日始/日终面板）、GameOver（结算面板）

### 核心玩法
- 每天5个时间单位，每个单位选择一种运营动作（数值调整/活动运营/体验优化/内容更新）
- 每个动作对不同玩家群体满意度产生差异化影响，同时影响KPI进度
- 7天为一局，每日KPI未达标累积"压力值"，压力满100被辞退
- 核心乐趣：氪金大佬 vs 白嫖党的天然对立，玩家必须被迫取舍

### 动作设计（11种）
包括加强免费/付费角色、调整平衡、充值/签到/PVP活动、减负优化、界面优化、推出新付费/免费内容等
